Дневник погоды
Автор
Александров Алексей

Краткое описание программы
Программа «Дневник погоды» — это графическое приложение с пользовательским интерфейсом для ведения ежедневных записей о погодных условиях. Приложение позволяет пользователю добавлять записи с указанием даты, температуры, текстового описания погоды и наличия осадков. Все записи отображаются в удобной таблице, поддерживается фильтрация по дате (точное совпадение) и по температуре (отображение записей с температурой выше заданного порога). Данные могут быть сохранены в JSON-файл и загружены из него для дальнейшего использования. Программа проверяет корректность формата даты, числовое значение температуры и заполнение описания, выводя понятные сообщения об ошибках при неверном вводе.

Инструкция по запуску
Требования к системе
Операционная система: Windows, macOS или Linux

Установленный интерпретатор Python версии 3.6 или выше

Зависимости
Для работы программы требуются только стандартные библиотеки Python, которые входят в базовую установку:

tkinter — для создания графического интерфейса (обычно уже установлен)

json — для работы с файлами JSON

os — для операций с файловой системой

datetime — для проверки формата даты

Дополнительных сторонних библиотек устанавливать не требуется.

Пошаговая инструкция по запуску
Скачайте файл программы
Сохраните файл main.py в отдельную папку на вашем компьютере (например, C:\weather_journal\ или ~/Documents/weather_journal/).

Откройте терминал или командную строку

Windows: нажмите Win + R, введите cmd и нажмите Enter

macOS: откройте Terminal через Launchpad или Spotlight

Linux: откройте Terminal через меню приложений

Перейдите в папку с программой
Используйте команду cd для перехода в каталог, где находится main.py.
Пример:

cd C:\weather_journal
или

cd ~/Documents/weather_journal
Запустите программу
Выполните следующую команду:

python main.py
Если на вашем компьютере установлено несколько версий Python, возможно, потребуется использовать:

python3 main.py
Ожидайте открытия окна приложения
После выполнения команды должно появиться графическое окно «Дневник погоды» с полями для ввода, таблицей и кнопками управления.

Возможные проблемы при запуске
Проблема	Решение
python не распознан как внутренняя или внешняя команда	Python не добавлен в PATH. Переустановите Python, отметив опцию «Add Python to PATH»
No module named tkinter	На некоторых системах Linux tkinter устанавливается отдельно: sudo apt-get install python3-tk
Окно открывается и сразу закрывается	Запустите программу через терминал, чтобы увидеть текст ошибки
Примеры использования и работы программы
Пример 1: Добавление новой записи о погоде
Действия пользователя:

В поле «Дата» оставлено значение по умолчанию 2026-05-05

В поле «Температура» введено 18.5

В поле «Описание» введено Солнечно, легкий ветерок

Чекбокс «Осадки» оставлен неотмеченным

Нажата кнопка «Добавить запись»

Результат:
В таблице появляется новая строка с данными:
2026-05-05 | 18.5 | Солнечно, легкий ветерок | Нет
Появляется всплывающее сообщение «Успех | Запись добавлена»

Пример 2: Попытка добавить запись с ошибками ввода (негативный тест)
Действия пользователя:

В поле «Дата» введено 05.05.2026 (неверный формат)

В поле «Температура» введено очень тепло

Поле «Описание» оставлено пустым

Нажата кнопка «Добавить запись»

Результат:
Программа последовательно выдаёт три сообщения об ошибках после каждой попытки:

«Ошибка | Неверный формат даты. Используйте ГГГГ-ММ-ДД»

После исправления даты: «Ошибка | Температура должна быть числом»

После исправления температуры: «Ошибка | Описание не может быть пустым»
Запись не добавляется до тех пор, пока все поля не будут заполнены корректно.

Пример 3: Фильтрация записей по температуре
Исходные данные в таблице:

Дата	Температура	Описание	Осадки
2026-05-01	5	Облачно	Да
2026-05-02	12	Дождливо	Да
2026-05-03	20	Ясно	Нет
2026-05-04	8	Туман	Нет
Действия пользователя:

В поле «Фильтр по температуре (> °C)» введено 10

Нажата кнопка «Применить» (под фильтром температуры)

Результат:
Таблица отображает только записи с температурой выше 10°C:

Дата	Температура	Описание	Осадки
2026-05-02	12	Дождливо	Да
2026-05-03	20	Ясно	Нет
Появляется сообщение «Информация | Записей с температурой выше 10°C не найдено» не появляется, так как записи найдены.

Пример 4: Фильтрация по дате
Действия пользователя:

В поле «Фильтр по дате» введено 2026-05-03

Нажата кнопка «Применить» (под фильтром даты)

Результат:
Таблица отображает только одну запись за 3 мая 2026 года.

Пример 5: Сохранение всех записей в JSON-файл
Действия пользователя:

Нажата кнопка «Сохранить в JSON»

В появившемся диалоговом окне выбран каталог и указано имя файла мой_дневник.json

Нажата кнопка «Сохранить»

Результат:
Создан файл мой_дневник.json со следующим содержимым:

json
[
    {
        "date": "2026-05-01",
        "temperature": 5,
        "description": "Облачно",
        "precipitation": "Да"
    },
    {
        "date": "2026-05-02",
        "temperature": 12,
        "description": "Дождливо",
        "precipitation": "Да"
    },
    {
        "date": "2026-05-03",
        "temperature": 20,
        "description": "Ясно",
        "precipitation": "Нет"
    }
]
Появляется сообщение: «Успех | Данные сохранены в C:/.../мой_дневник.json»

Пример 6: Загрузка данных из JSON-файла
Действия пользователя:

Нажата кнопка «Загрузить из JSON»

В диалоговом окне выбран ранее сохранённый файл мой_дневник.json

Нажата кнопка «Открыть»

Результат:
Программа загружает записи из файла, отображает их в таблице и выводит сообщение: «Успех | Данные загружены из C:/.../мой_дневник.json»

Пример 7: Сброс фильтров
Действия пользователя:
После применения любого фильтра нажата кнопка «Сбросить» (соответствующая активному фильтру — под датой или под температурой)

Результат:
Поле ввода фильтра очищается, таблица снова показывает все записи из текущего набора данных.
